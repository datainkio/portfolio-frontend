# Frontend Architecture (AIX-focused)

> Source is the authority. Generated assets (\_site/) are never committed; builds always run from source.

## Repo boundaries

- This repo is the frontend (11ty + Tailwind + GSAP). The backend repo (Sanity Studio + schema) lives as a sibling at `../backend/`, not inside this folder.
- `data/sanity/` here is the read-side only: client, queries, and fetch helpers used by 11ty. Schema and Studio changes belong in the backend repo.

## Top-level layout (source vs tooling vs generated)

- `ia/` — Content entrypoints (frontmatter routes; Eleventy `input`).
- `views/` — Templates and components (atomic design; Eleventy `includes`).
- `js/` — Browser runtime (choreography, effects, displays, preloader, utils). No build outputs here; passthrough happens in Eleventy.
- `styles/` — Tailwind entry and custom CSS. `styles/colors.css` and `styles/typography/fontFamilies.css` are generated by design sync.
- `assets/` — Source assets copied to `_site/assets/` during build. Keep only inputs here.
- `eleventy/` — Modular Eleventy config (collections, filters, shortcodes, plugins, services).
- `data/sanity/` — Sanity client, fetch helpers, and queries (read-side only).
- `figma/` — Design token services and fetchers (generate CSS tokens).
- `scripts/` — Build and automation scripts (CSS build, JS bundle, Figma sync, scaffolding).
- `site.json` — Global site config + CMS defaults.
- `docs/` — Human/AI docs. AI prompts live in `docs/ai/`. Historical/recap docs live in `docs/_archive/`.
- Generated/noise (ignored): `_site/`, `.cache/`, `logs/`, `node_modules/`.

## Rules of placement

- Runtime code (templates, JS, CSS) stays in `views/`, `js/`, `styles/`. Do not place runtime code in `scripts/` or `docs/`.
- Tooling stays in `scripts/`, `eleventy/`, and root config files (`.eleventy.js`, `tailwind.config.js`, `postcss.config.js`).
- Generated outputs are ignored and never edited: `_site/`, `.cache/`, `logs/`, `styles/colors.css`, `styles/typography/fontFamilies.css`.
- Experiments/prompts belong in `docs/ai/` and should be short-lived.

## Add something new?

- New page/route: `ia/` entry + any new components under `views/`.
- New client behavior: `js/` (follow choreography/sections patterns); ensure DOM hooks exist in templates.
- New styles: `styles/` (respect `styles/main.css` import order). Regenerate design tokens before CSS build when tokens change.
- New CMS content: add/update queries in `data/sanity/queries.js`.

## Build truth

- Builds must run from source; CI/deploy should execute `npm run build` (or `npm run quick` for fast paths).
- Full build order: `clean → build:design → build:css → build:js → build:11ty`. Skipping `build:design` will compile but produce a site without Figma-derived tokens.
- `_site/` is purely an output directory and should not be read by Copilot or committed.
